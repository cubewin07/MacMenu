# Architecture Decision Records (ADRs)

An ADR captures one significant decision: context, options, choice, and consequences. ADRs are numbered and immutable — supersede rather than rewrite.

## Format
- **Status** — Proposed / Accepted / Superseded by ADR-XXXX
- **Context** — forces and constraints
- **Decision** — what we chose
- **Options considered** — alternatives and why they lost
- **Consequences** — the tradeoffs we accept
